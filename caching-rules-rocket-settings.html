Caching Rules & WP Rocket Settings

Cloudflare cache rules and WP Rocket configuration for teambuildconsultancy.co.uk

Cloudflare Cache Rules

Add these rules under Rules → Cache Rules in the Cloudflare dashboard. Order matters — static cache rule first.

Rule 1: Cache Static Assets

Edge TTL: 30 days
(http.request.uri.path contains ".jpg" or http.request.uri.path contains ".jpeg" or http.request.uri.path contains ".gif" or http.request.uri.path contains ".png" or http.request.uri.path contains ".webp" or http.request.uri.path contains ".avif" or http.request.uri.path contains ".css" or http.request.uri.path contains ".js" or http.request.uri.path contains ".svg" or http.request.uri.path contains ".woff" or http.request.uri.path contains ".woff2" or http.request.uri.path contains ".eot" or http.request.uri.path contains ".ttf" or http.request.uri.path contains ".ico" or http.request.uri.path contains ".mp4" or http.request.uri.path contains ".pdf" or http.request.uri.path contains ".zip" or http.request.uri.path contains ".gz")
Cache status: Eligible for cache
Edge TTL: 30 days
Browser TTL: 30 days

Rule 2: Bypass Admin & Login

Bypass
(starts_with(http.request.uri.path, "/wp-admin/") or http.request.uri.path eq "/wp-login.php")
Cache status: Bypass

Rule 3: Bypass REST API & XML-RPC

Bypass
(starts_with(http.request.uri.path, "/wp-json/") or http.request.uri.path eq "/xmlrpc.php" or http.request.uri.path contains "rest_route=")
Cache status: Bypass

Rule 4: Bypass Logged-in Users

Bypass
(http.request.cookies contains "wordpress_logged_in_")
Cache status: Bypass

Optional — wp-config.php

Add before /* That's all */ in wp-config.php:

define('WP_AUTO_UPDATE_CORE', false);
define('DISABLE_WP_CRON', true);
Note: WP Rocket manages .htaccess caching rules automatically — do not edit manually.
Important: After applying these changes, thoroughly test the site to verify everything works — check front-end pages, admin login, cache purge, and confirm no broken functionality.
